WisiToken Overview

WisiToken is an LALR parser generator and run-time. The grammar can be specified in a bison-like input file, or as Ada statements. The run-time parser supports generalized parsing (parallel parsers to handle conflicts) and syntax error recovery (inserting and deleting tokens to allow parsing to continue).

The primary motivation for supporting generalized parsing is to simplify maintaining the Emac Ada indentation engine, which uses a parser generated ty WisiToken. The Ada grammar specified in the Ada Language Reference manual is not LALR(1); it has shift/reduce and reduce/reduce conflicts. Modifying the grammar to be LALR(1) is hard, and complicates adding future changes to the language.. Using a generalized LALR parser to handle the conflicts allows keeping the grammar (mostly) in its original form.

WisiToken provides a simple regular-expression based lexer, used mostly in unit tests. It supports a generic lexer interface, with one instance providing the AdaFlex lexer, and another the Emacs wisi lexer. Other lexers can be added easily.

WisiToken is distributed under GPL version 3, with the GNAT modification that allows use in non-GPL projects.

WisiToken is a significant rewrite of OpenToken, deleting support for recursive-descent, adding syntax error correction in the parser, and focusing on parsers produced by wisi-generate and used by Emacs.

current version: 1.0

WisiToken may be obtained in several ways:

  • source Gnu tar bzip2 wisitoken-1.0a.tar.bz2
  • monotone server www.ada-france.org, branch org.wisitoken. The ada-france server is maintained by Ludovic Brenta

WisiToken uses Ada 2012 containers, so it requires an Ada 2012 compiler.

WisiToken is currently maintained by Stephen Leake. Submit bugs directly to Stephen. Discussion about WisiToken is on the newsgroup comp.lang.ada, or the Emacs Ada mode mailing list.

OpenToken has been tested on the following operating systems/compilers:

  • Windows 8 64 bit / GNAT GPL 2016 32 bit

History

Version 1.0

  • Major rewrite/renaming of OpenToken.
  • Refactor packages to support error correction in the run-time parser, and to support LR1 parsers.
  • Delete support for recursive descent parsers; broken by the refactor, not worth fixing.
  • Extend the LALR parser to support error recovery.
  • Add support for Aflex and other lexers - requires Aflex 1.5
  • Add support for the Emacs wisi lexer.
